What it does

Built as a class project for Usable Security & Privacy. Designed to be simple, transparent, and safe by default.

⚡

Auto-rejects when it's clear

Looks for obvious top-level options like "Reject", "Reject All", "Decline", or "Deny" and clicks them for you.

🛡️

Safe fallback

If there's no direct reject option, it opens the privacy or cookie settings panel and stops — so you stay in control.

🔍

Watches the page

Some banners load late. The extension keeps watching for a short window so it can act once the banner finally appears.

💻

Runs locally

No backend, no account, no analytics. Everything runs inside your browser.

📦

Manifest V3

Built on Chrome's current extension platform with a small, focused permission set.

🧪

Easy to inspect

Exposes a debug attribute on the page so you can see what it did: rejected, opened-manual-choice, or manual-choice.

How it works

A simple decision flow that prioritizes safety over speed.

1

Look for a clear "Reject"

Scans buttons and links for keywords like reject, decline, deny, or only necessary. If it finds one, it clicks it.

2

Otherwise, open settings

If there's no direct reject button, it looks for "Manage Cookies", "Privacy Choices", "Customize", and similar entries and opens that panel.

3

Hand it back to you

Inside complex second-layer modals, the extension stops and lets the user make the final choice. No risky guessing.

Install in under a minute

The extension is unpacked / developer-mode for now since it's a class project.

  1. Open the GitHub repo and click Code → Download ZIP, then unzip it. Or git clone it.
  2. In Chrome, visit chrome://extensions.
  3. Turn on Developer mode in the top-right corner.
  4. Click Load unpacked and select the folder containing manifest.json.
  5. Open the extension's details and set Site access to On all sites.
  6. Visit any site with a cookie banner and watch it go.

Full instructions, including how to update and debug, are in INSTALL.md.

Where it falls short (we're being honest)

A class project, not a polished product. These are the things we know don't work yet.

Custom HTML & vendor SDKs

Many cookie banners are built with shadow DOM, iframes, or vendor SDKs that hide the buttons from a simple text scan.

Second-layer modals

Once the privacy panel is open, the extension does not auto-toggle off optional cookies or submit the form for you.

Non-English text

Keyword matching is in English. Banners in other languages or with unusual wording will likely be missed.

Sites change

If a site updates its consent UI, behavior that worked yesterday might break today.

🔒

Privacy: it stays on your machine

The extension runs locally inside your browser. There is no remote server, no account system, and no telemetry. It only interacts with cookie banners on the page you're already visiting.

Common questions

A few things people ask when we demo this.

Will this work on every website?

No. Cookie banners are wildly inconsistent. The extension handles common patterns where a clear "Reject" or "Manage" button exists in plain HTML. Sites that hide their banner inside shadow DOM, iframes, or custom widgets may not work yet.

Why doesn't it just click everything off in the settings panel?

Because that's where things get risky. Second-layer settings differ from site to site, and a wrong click could accept cookies instead of rejecting them. We chose to stop at the panel and let the user finish.

Is it safe to install?

The code is open source on GitHub — you can read every file. It uses activeTab, scripting, tabs, and webNavigation permissions to detect and click banner buttons. It does not send data anywhere.

How can I tell what the extension did on a page?

Open DevTools and check document.documentElement.dataset.autoCookieRejectorState. Possible values are rejected, opened-manual-choice, and manual-choice.

Is this on the Chrome Web Store?

Not yet. It's a class project for Usable Security & Privacy, installed via Chrome's Developer Mode for now.
